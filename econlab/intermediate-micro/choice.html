The Consumer’s Choice Problem

The picture

The consumer wants to reach the highest indifference curve they can afford. Geometrically, the optimum is where the indifference curve just touches the budget line — they are tangent.

At the tangency, the slope of the indifference curve (the marginal rate of substitution, MRS) equals the slope of the budget line (the price ratio):

\[ \text{MRS}_{12} = \frac{p_1}{p_2}. \]

That’s it. Every interior consumer choice problem reduces to this condition, plus the budget constraint \(p_1 x_1 + p_2 x_2 = m\).

Cobb-Douglas: the closed-form workhorse

If preferences are Cobb-Douglas — \(u(x_1, x_2) = x_1^\alpha x_2^{1-\alpha}\) with \(0 < \alpha < 1\) — then the MRS is

\[ \text{MRS}_{12} = \frac{\partial u/\partial x_1}{\partial u/\partial x_2} = \frac{\alpha}{1 - \alpha} \cdot \frac{x_2}{x_1}. \]

Setting MRS equal to the price ratio and using the budget constraint gives the optimal bundle:

\[ \boxed{\; x_1^* = \alpha \cdot \frac{m}{p_1}, \qquad x_2^* = (1 - \alpha) \cdot \frac{m}{p_2}. \;} \]

Two things worth noticing:

  1. The consumer spends a fixed share of income on each good. Spending on \(x_1\) is \(p_1 x_1^* = \alpha m\). Spending on \(x_2\) is \((1 - \alpha) m\). That’s why Cobb-Douglas is so popular as a workhorse model — the math is one line.
  2. Demand for \(x_1\) doesn’t depend on \(p_2\). Cross-price effects are zero. That’s not a feature of the world — it’s a feature of Cobb-Douglas. More flexible utility functions (CES, quasilinear) break this.

library(shiny)

ui <- fluidPage(
  titlePanel("Optimal Choice with Cobb-Douglas Preferences"),

  sidebarLayout(
    sidebarPanel(
      width = 4,
      sliderInput("alpha", "Preference weight α (taste for good 1):",
                  min = 0.1, max = 0.9, value = 0.5, step = 0.05),
      sliderInput("m",  "Income (m):",            min = 20, max = 200, value = 100, step = 10),
      sliderInput("p1", "Price of good 1 (p₁):",  min =  1, max =  20, value =   5, step = 1),
      sliderInput("p2", "Price of good 2 (p₂):",  min =  1, max =  20, value =   5, step = 1),
      hr(),
      checkboxInput("show_other_ics",
                    "Show several indifference curves",
                    value = TRUE),
      htmlOutput("readout")
    ),
    mainPanel(
      width = 8,
      plotOutput("choice_plot", height = "520px")
    )
  )
)

server <- function(input, output, session) {

  bundle <- reactive({
    alpha <- input$alpha
    m  <- input$m; p1 <- input$p1; p2 <- input$p2
    x1 <- alpha * m / p1
    x2 <- (1 - alpha) * m / p2
    u_opt <- x1^alpha * x2^(1 - alpha)
    list(alpha = alpha, m = m, p1 = p1, p2 = p2,
         x1 = x1, x2 = x2, u_opt = u_opt)
  })

  output$choice_plot <- renderPlot({
    b <- bundle()

    x1_int <- b$m / b$p1
    x2_int <- b$m / b$p2

    xmax <- max(40, x1_int) * 1.05
    ymax <- max(40, x2_int) * 1.05

    par(mar = c(4.2, 4.5, 1, 1))
    plot(NA, xlim = c(0, xmax), ylim = c(0, ymax),
         xlab = expression(x[1]), ylab = expression(x[2]),
         main = "")

    # Indifference curve at optimum: x2 = (u / x1^alpha)^(1/(1-alpha))
    ic_x <- seq(0.1, xmax * 1.5, length.out = 600)
    ic_y_opt <- (b$u_opt / ic_x^b$alpha)^(1 / (1 - b$alpha))

    # Optional ladder of indifference curves.
    if (isTRUE(input$show_other_ics)) {
      for (frac in c(0.5, 0.75, 1.25)) {
        u <- b$u_opt * frac
        y <- (u / ic_x^b$alpha)^(1 / (1 - b$alpha))
        lines(ic_x, y, col = adjustcolor("#7f8c8d", 0.5), lwd = 1.2, lty = 3)
      }
    }

    # Highlighted indifference curve through optimum.
    lines(ic_x, ic_y_opt, col = "#27ae60", lwd = 2.4)

    # Budget line and shaded affordable set.
    polygon(c(0, x1_int, 0), c(0, 0, x2_int),
            col = adjustcolor("#3498db", 0.12), border = NA)
    segments(0, x2_int, x1_int, 0, col = "#185FA5", lwd = 3)

    # Optimum point.
    points(b$x1, b$x2, pch = 19, col = "#c0392b", cex = 1.8)
    text(b$x1, b$x2,
         labels = sprintf("  (%.1f, %.1f)", b$x1, b$x2),
         pos = 4, col = "#c0392b", cex = 1.0)

    legend("topright",
           legend = c("Budget line", "Indifference curve through optimum",
                      if (isTRUE(input$show_other_ics)) "Other indifference curves" else NULL,
                      "Optimal bundle"),
           col    = c("#185FA5", "#27ae60",
                      if (isTRUE(input$show_other_ics)) "#7f8c8d" else NULL,
                      "#c0392b"),
           lwd    = c(3, 2.4, if (isTRUE(input$show_other_ics)) 1.2 else NULL, NA),
           lty    = c(1, 1, if (isTRUE(input$show_other_ics)) 3 else NULL, NA),
           pch    = c(NA, NA, if (isTRUE(input$show_other_ics)) NA else NULL, 19),
           bty = "n", cex = 0.9)
  })

  output$readout <- renderUI({
    b <- bundle()
    share1 <- b$alpha
    share2 <- 1 - b$alpha
    HTML(sprintf(paste(
      "<div style='margin-top:10px;font-size:13px;line-height:1.7;'>",
      "<b>Optimal bundle:</b> x₁* = %.2f, x₂* = %.2f<br>",
      "<b>Spending on good 1:</b> p₁·x₁* = %.1f (%.0f%% of income)<br>",
      "<b>Spending on good 2:</b> p₂·x₂* = %.1f (%.0f%% of income)<br>",
      "<b>MRS at optimum:</b> %.2f &nbsp; <b>p₁/p₂:</b> %.2f<br>",
      "<b>Utility achieved:</b> %.2f",
      "</div>"
    ),
    b$x1, b$x2,
    b$p1 * b$x1, 100 * share1,
    b$p2 * b$x2, 100 * share2,
    (b$alpha / (1 - b$alpha)) * (b$x2 / b$x1),
    b$p1 / b$p2,
    b$u_opt))
  })
}

shinyApp(ui, server)

Things to try.

  • Move \(\alpha\) from 0.1 to 0.9 with everything else fixed. The spending share on good 1 tracks \(\alpha\) exactly. The indifference curves rotate; the optimum slides along the budget line.
  • Hold \(\alpha = 0.5\) and double \(p_1\). Notice the spending shares don’t change — that’s the Cobb-Douglas feature. But the quantity of \(x_1\) halves.
  • Try a corner case — set \(\alpha = 0.9, p_1 = 1, p_2 = 20\). The optimum sits almost entirely on the \(x_1\) axis; the consumer barely consumes \(x_2\).
  • Check MRS = price ratio in the readout — they should be equal at every interior optimum. If you ever see them disagree, something is at a corner.

When the tangency rule fails

The “MRS = price ratio” condition is the interior first-order condition. It can fail in two ways:

Corner solutions. If preferences are strong enough (or the price gap large enough), the consumer wants zero of one good. Then the constraint \(x_i \geq 0\) binds and the tangency condition becomes an inequality. Perfect substitutes \(u = ax_1 + bx_2\) are the canonical case: the consumer buys only the cheaper-per-utility good unless the ratio \(a/b\) exactly matches \(p_1/p_2\).

Kinked indifference curves. Perfect complements \(u = \min(x_1, x_2)\) have no well-defined MRS at the kink, so tangency is meaningless. The optimum sits at the kink where \(x_1 = x_2\) — derivable directly from the constraint without any calculus.

These cases all reduce to the same idea: the consumer reaches the highest affordable indifference curve. Tangency is one way that happens; corners and kinks are others.

Did you know?

  • The “tangency” picture is the Lagrangian in disguise. Maximizing \(u(x_1, x_2)\) subject to \(p_1 x_1 + p_2 x_2 = m\) gives the first-order conditions \(\partial u/\partial x_i = \lambda p_i\), which rearrange to \(\text{MRS} = p_1/p_2\).
  • The Lagrange multiplier \(\lambda\) at the optimum is the marginal utility of income — how much utility goes up if you give the consumer one extra dollar. That’s where the “value of an extra dollar” interpretation of \(\lambda\) comes from.
  • For Cobb-Douglas, you’ll often see the equivalent form \(u(x_1, x_2) = \alpha \log x_1 + (1 - \alpha) \log x_2\) — a monotonic transformation that doesn’t change choice but makes derivatives cleaner. The log form will reappear later in the Demand chapter.
