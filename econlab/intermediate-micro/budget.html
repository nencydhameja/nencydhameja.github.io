Budget Constraints

The picture

You have income \(m\) and you can buy two goods \(x_1, x_2\) at prices \(p_1, p_2\). The set of bundles you can afford is

\[ \{(x_1, x_2) \geq 0 : p_1 x_1 + p_2 x_2 \leq m\}. \]

The boundary of that set — where you spend every dollar — is the budget line:

\[ p_1 x_1 + p_2 x_2 = m. \]

Rewritten with \(x_2\) on the vertical axis:

\[ x_2 = \frac{m}{p_2} \;-\; \frac{p_1}{p_2} x_1. \]

Three things to read off this equation:

  • Vertical intercept \(m/p_2\) — the most \(x_2\) you can buy if you spend everything on \(x_2\).
  • Horizontal intercept \(m/p_1\) — the most \(x_1\) you can buy if you spend everything on \(x_1\).
  • Slope \(-p_1/p_2\) — the rate at which the market lets you trade \(x_2\) for \(x_1\). Give up one unit of \(x_1\), the market gives you \(p_1/p_2\) units of \(x_2\).

That’s the whole story. Everything else in consumer theory builds on top of this picture.

Three shifts to keep in mind

Change What happens to the budget line
Income \(m\) up Parallel shift outward (same slope)
Price \(p_1\) up Pivots inward around the \(x_2\) intercept
Both prices up by the same factor Same as income falling — the line shifts inward in parallel
Composite price shift (\(p_1, p_2\) both change by the same factor) Pivots around no fixed point unless you also adjust \(m\)

The third row is the homogeneity of degree zero result: if all prices and income scale by the same factor, the budget set is unchanged. Only relative prices matter for what you can buy.

library(shiny)

ui <- fluidPage(
  titlePanel("Budget Line Explorer"),

  sidebarLayout(
    sidebarPanel(
      width = 4,
      sliderInput("m",  "Income (m):",            min =  20, max = 200, value = 100, step = 10),
      sliderInput("p1", "Price of good 1 (p₁):",  min =   1, max =  20, value =   5, step = 1),
      sliderInput("p2", "Price of good 2 (p₂):",  min =   1, max =  20, value =  10, step = 1),
      hr(),
      checkboxInput("show_old", "Show the previous budget line for comparison", value = TRUE),
      htmlOutput("readout")
    ),
    mainPanel(
      width = 8,
      plotOutput("budget_plot", height = "480px")
    )
  )
)

server <- function(input, output, session) {

  # Cache the previous parameters so we can overlay the previous line.
  prev <- reactiveValues(m = 100, p1 = 5, p2 = 10)

  observeEvent(list(input$m, input$p1, input$p2), {
    isolate({
      prev$m_old  <- prev$m
      prev$p1_old <- prev$p1
      prev$p2_old <- prev$p2
      prev$m  <- input$m
      prev$p1 <- input$p1
      prev$p2 <- input$p2
    })
  })

  output$budget_plot <- renderPlot({
    m  <- input$m;  p1 <- input$p1; p2 <- input$p2
    x1_intercept <- m / p1
    x2_intercept <- m / p2

    xmax <- max(220 / 5, x1_intercept) * 1.1
    ymax <- max(220 / 5, x2_intercept) * 1.1

    par(mar = c(4.2, 4.5, 1, 1))
    plot(NA, xlim = c(0, xmax), ylim = c(0, ymax),
         xlab = expression(x[1]), ylab = expression(x[2]),
         main = "")

    # Shade the affordable set.
    polygon(c(0, x1_intercept, 0), c(0, 0, x2_intercept),
            col = adjustcolor("#3498db", 0.18), border = NA)

    # The previous budget line (faded), if requested.
    if (isTRUE(input$show_old) &&
        (prev$m_old != m || prev$p1_old != p1 || prev$p2_old != p2)) {
      segments(0, prev$m_old / prev$p2_old,
               prev$m_old / prev$p1_old, 0,
               col = "#bdc3c7", lwd = 2, lty = 2)
    }

    # Current budget line.
    segments(0, x2_intercept, x1_intercept, 0,
             col = "#185FA5", lwd = 3)

    # Intercept labels.
    points(c(x1_intercept, 0), c(0, x2_intercept), pch = 19, col = "#185FA5")
    text(x1_intercept, 0, labels = sprintf("m/p₁ = %.1f", x1_intercept),
         pos = 3, cex = 0.9, col = "#185FA5")
    text(0, x2_intercept, labels = sprintf("m/p₂ = %.1f", x2_intercept),
         pos = 4, cex = 0.9, col = "#185FA5")

    legend("topright",
           legend = c("Current budget line", "Affordable bundles",
                      if (isTRUE(input$show_old)) "Previous budget line" else NULL),
           col    = c("#185FA5", adjustcolor("#3498db", 0.4),
                      if (isTRUE(input$show_old)) "#bdc3c7" else NULL),
           lwd    = c(3, NA, if (isTRUE(input$show_old)) 2 else NULL),
           lty    = c(1, NA, if (isTRUE(input$show_old)) 2 else NULL),
           pch    = c(NA, 15, if (isTRUE(input$show_old)) NA else NULL),
           pt.cex = 1.6,
           bty = "n", cex = 0.9)
  })

  output$readout <- renderUI({
    m  <- input$m; p1 <- input$p1; p2 <- input$p2
    HTML(sprintf(paste(
      "<div style='margin-top:10px;font-size:13px;line-height:1.7;'>",
      "<b>Budget line:</b> %d · x₁ + %d · x₂ = %d<br>",
      "<b>Slope (−p₁/p₂):</b> %.2f<br>",
      "<b>Max x₁:</b> %.1f &nbsp; <b>Max x₂:</b> %.1f",
      "</div>"
    ), p1, p2, m, -p1 / p2, m / p1, m / p2))
  })
}

shinyApp(ui, server)

Things to try.

  • Raise income \(m\): the line slides outward without changing its slope. Affordable region grows; trade-off between goods is unchanged.
  • Raise \(p_1\): the horizontal intercept moves in, the vertical one stays put. The line pivots. Good 1 is more expensive relative to good 2, so the slope steepens.
  • Double both prices: same as halving income. The line shifts in parallel, even though no “price ratio” has changed. Relative prices alone don’t determine what you can afford — your real income does too.
  • Try \(p_1 = p_2\): the slope is −1. One unit of either good costs the same; the budget line cuts the axes at \(m/p\) in both directions.

A few subtleties

Composite goods. Real consumers buy thousands of goods. The two-good model isn’t a claim that there are two goods — it’s a claim that you can lump “everything else” into a composite good \(x_2\) with price normalized to \(1\). Then \(m - p_1 x_1\) is what’s left over for everything else. That makes \(x_1\) the choice variable of interest and keeps the geometry two-dimensional.

Taxes, subsidies, rationing. Each of these changes the budget set’s shape:

  • A quantity tax of \(t\) on good 1 acts like a higher price: \(p_1\) becomes \(p_1 + t\).
  • A value tax (\(\tau\)) acts like a price multiplier: \(p_1\) becomes \((1 + \tau) p_1\).
  • A lump-sum tax of \(T\) acts like an income drop: \(m\) becomes \(m - T\).
  • A food-stamp subsidy (lower price on \(x_1\) up to a quantity cap) kinks the budget line.
  • A ration (hard quantity ceiling on \(x_1\)) truncates the budget set with a vertical line.

Each of these is one or two extra strokes on the same picture. Try sketching them.

Endowment-based income. What if you don’t have money — you have stuff? You start with \((\omega_1, \omega_2)\) units of the two goods. Then your “income” at prices \((p_1, p_2)\) is \(m = p_1 \omega_1 + p_2 \omega_2\), and the budget line passes through your endowment. We’ll come back to this when we cover Buying and Selling and Exchange.

Did you know?

  • The two-good budget line is sometimes called the trade-off line or price line. Different textbooks emphasize different framings: as a constraint (what you can afford), as an opportunity cost schedule (how much \(x_2\) a unit of \(x_1\) costs you), or as a market trade rate. All three readings are the same line.
  • The standard “all-or-nothing” framing — your budget is exactly \(m\), no borrowing, no saving — drops in Intertemporal Choice, where the budget constraint becomes a present-value constraint over two or more periods.
